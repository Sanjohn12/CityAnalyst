# Contributing to accessmap-react

First off, thank you for considering contributing to `accessmap-react`! It's people like you who make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue on GitHub. Include:
- A clear description of the bug.
- Steps to reproduce.
- Your environment (browser, React version, etc.).

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. Provide:
- A clear summary of the suggestion.
- Why this would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`npm test`).
5. Make sure your code lints (`npm run lint`).

## Development Setup

```bash
# Clone the repository
git clone https://github.com/Sanjohn12/CityAnalyst.git
cd accessmap-react

# Install dependencies
npm install

# Start development server
npm run dev
```

## Code Style
- Use functional components and hooks.
- Follow the existing ESLint configuration.
- Write JSDoc comments for all public APIs.
